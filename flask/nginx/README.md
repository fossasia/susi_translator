# Nginx Reverse Proxy & SSL Configuration

This directory contains the necessary configuration and scripts to run Nginx as a secure reverse proxy for the Susi Translator Flask application. It automatically generates a self-signed SSL certificate on startup, routes HTTP traffic to HTTPS, and forwards API requests to the backend Flask service.

## Architecture

Nginx sits in front of the Flask (`web`) container in the Docker Compose network. Its primary responsibilities are:

1. **SSL Termination**: Serving the application over HTTPS using a self-signed certificate.
2. **Static File Serving**: Efficiently serving static assets directly, bypassing the Flask application.
3. **Request Routing & Proxying**: Forwarding backend API requests and handling specific timeout requirements for long-running endpoints (e.g., model warmup and SSE streaming).

## File Breakdown

### `Dockerfile.nginx`

A lightweight Alpine-based Nginx Dockerfile.

- **What it does**: Builds the custom Nginx image, installs `openssl`, and copies the `generate_ssl.sh` script to `/docker-entrypoint.d/`.
- **Why `/docker-entrypoint.d/`?**: Any executable scripts placed in this directory are automatically run by the official Nginx image's entrypoint before the Nginx daemon starts. This guarantees the SSL certificates are generated right before Nginx attempts to read them.

### `generate_ssl.sh`

A shell script responsible for generating a self-signed SSL certificate.

- **What it does**: Checks if an SSL certificate and private key exist at `/etc/nginx/ssl`. If not, it uses `openssl` to generate a 2048-bit RSA self-signed certificate valid for 365 days.
- **How it's used**: Executed automatically on container startup by the Nginx entrypoint. It prevents manual certificate generation overhead during local development or simple deployments.

### `nginx.conf`

The main Nginx configuration file.

- **What it does**:
  - Sets up an `upstream` block pointing to the `web:5040` Flask service.
  - Redirects all incoming `HTTP` traffic (port 80) to `HTTPS` (port 443).
  - Configures `HTTPS` settings using the certificates generated by `generate_ssl.sh`.
  - Serves static files directly from the `/app/static/` directory.
  - Proxies requests to `/api/v1/translate/stream` without buffering, ensuring real-time Server-Sent Events (SSE) reach the client immediately.
  - Proxies requests to `/api/v1/translate/configure` with extended timeouts (300s) to accommodate potentially long-running ML model warmup operations.
  - Acts as a default proxy for all other requests.

## How It Fits Together (Developer Workflow)

When you run `docker-compose up`:

1. Docker builds the `nginx` service using `Dockerfile.nginx`.
2. The Nginx container starts up and its entrypoint executes `generate_ssl.sh`.
3. The self-signed certificate and key are created in `/etc/nginx/ssl`.
4. Nginx loads `nginx.conf`, which references those certificates.
5. The Flask app (`web` service) starts up.
6. The user can now securely access the application via `https://localhost` without dealing with manual certificate setup.
